# Project Brief: Retail Pulse Analytics Suite

## Overview
**Retail Pulse** is a comprehensive, end-to-end data analytics portfolio project designed to demonstrate the exact skills required for the Entry-Level Data Analyst role at Camping World. It simulates a retail environment where we track sales, customer behavior, and website traffic.

## Objective
To build a functional data pipeline and analytics dashboard that transforms raw transaction data into actionable business insights.

## Alignment with Job Requirements
| Camping World Requirement | Retail Pulse Project Feature |
|---------------------------|------------------------------|
| **Collect, clean, prepare data** | `data_generator.py`: Generates dirty data and an ETL process to clean it. |
| **SQL & Database Management** | SQLite database implementation with complex schemas and relations. |
| **Python & R Programming** | Core logic written in Python (Pandas, NumPy). |
| **Data Visualization / Dashboards** | Interactive Streamlit Dashboard (simulating Power BI reports). |
| **KPI Tracking (Traffic, Conversion)** | Dedicated "Web Analytics" module tracking pathing and conversion rates. |
| **Customer Segmentation** | Implementation of RFM (Recency, Frequency, Monetary) analysis. |

## Tech Stack
- **Language**: Python 3.9+
- **Database**: SQLite
- **Libraries**: Pandas, Faker (for data gen), Plotly (viz), Streamlit (UI).

## Key Deliverables
1.  **Automated Data Pipeline**: Script to generate and load data.
2.  **SQL Analysis Layer**: Library of SQL queries answering business questions.
3.  **Interactive Dashboard**: Web interface for stakeholders to view KPIs.
